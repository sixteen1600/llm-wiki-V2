# Data Index

本頁用於整理 `wiki/data_notes/` 中的資料筆記。

`data_index.md` 的重點不是保存完整資料內容，而是提供資料型筆記的導覽入口，幫助快速查詢：

- 目前整理過哪些資料主題
- 每個資料主題的來源、用途與敏感性
- 資料和哪些任務、專案、工具或流程有關
- 資料是否涉及欄位理解、資料品質、清理方式或轉換流程
- 哪些資料主題仍需要確認、抽象化或補成正式 wiki 筆記

---

## 1. Index 說明

本 index 對應以下資料夾：

| 類型 | 位置 | 說明 |
|---|---|---|
| 資料筆記 | `wiki/data_notes/` | 實習期間接觸到的資料來源、資料結構、欄位理解、資料品質、資料清理與資料轉換筆記 |

適合放入 `wiki/data_notes/` 的內容包括：

| 類型 | 範例 |
|---|---|
| 資料來源 | Excel、CSV、JSON、API 回傳資料、報表資料、系統匯出資料 |
| 資料結構 | 欄位概念、資料型態、主鍵、日期欄位、狀態欄位 |
| 資料品質 | 缺失值、重複值、異常值、格式不一致、編碼問題 |
| 資料處理 | 資料清理、欄位轉換、格式整理、資料合併 |
| 資料流程 | 資料輸入、處理、輸出、報表產生流程 |
| 資料限制 | 權限限制、敏感資訊、欄位定義未確認、資料更新頻率 |

## 5. 資料來源追蹤

本區塊用於追蹤資料主題最早出現在哪些原始筆記或學習紀錄中。

| 資料主題 | 首次出現日期 | 原始來源 | 對應 Log | 整理狀態 |
|---|---|---|---|---|
| `[[data_topic]]` | YYYY-MM-DD | `raw/imported_notes/YYYY/YYYY_MM_DD_Ddd.md` | `logs/daily/YYYY-MM-DD.md` | not_started / partial / completed |

### 使用規則

- 若資料主題來自每日原始筆記，應填入 `raw/imported_notes/` 來源。
- 若資料主題已整理成 daily log，應填入對應 `logs/daily/`。
- 若資料主題來自會議整理，應填入 `logs/meetings/`。
- 不要在本 index 中保存完整原始資料。
- 若來源包含敏感資訊，只保留抽象化描述與檔案位置。

## 6. 資料敏感性追蹤

本區塊用於檢查資料主題是否涉及敏感資訊。

| 資料主題 | 敏感性 | 風險類型 | 處理方式 | 確認狀態 |
|---|---|---|---|---|
| `[[data_topic]]` | none / abstracted / needs_review / sensitive | personal_data / policy_data / system_data / internal_field / business_rule / other | remove / abstract / mask / keep / pending | unchecked / reviewed / confirmed |

### 敏感性定義

| 敏感性 | 說明 |
|---|---|
| none | 不含敏感資訊，可直接整理成抽象資料筆記 |
| abstracted | 原始內容可能敏感，但已抽象化處理 |
| needs_review | 是否敏感仍需確認 |
| sensitive | 明確涉及敏感資訊，不應保存在 wiki 中 |

### 常見風險類型

| 風險類型 | 說明 |
|---|---|
| personal_data | 客戶個資、可識別個人資料 |
| policy_data | 保單、交易、契約或業務資料 |
| system_data | 內部系統名稱、網址、架構或權限資訊 |
| internal_field | 公司內部欄位名稱、資料表名稱或欄位代碼 |
| business_rule | 未公開的業務規則、決策邏輯或計算規則 |
| other | 其他需確認或抽象化的資訊 |

## 7. 資料結構與欄位追蹤

本區塊用於整理資料結構與欄位概念。  
此處只保存抽象欄位說明，不保存真實敏感欄位名稱或資料值。

| 資料主題 | 主要資料結構 | 重要欄位概念 | 資料型態 | 是否已確認 |
|---|---|---|---|---|
| `[[data_topic]]` | table / document / json / report / log / other |  | text / number / date / boolean / category / mixed | 是 / 否 / 待確認 |

### 欄位整理原則

- 可使用「日期欄位」、「狀態欄位」、「識別欄位」、「金額欄位」等抽象名稱。
- 不要保存真實客戶資料。
- 不要保存完整內部資料表名稱。
- 不要保存敏感欄位代碼。
- 若欄位定義尚未確認，應標記為「待確認」。

## 8. 資料品質問題追蹤

本區塊整理資料品質相關問題。

| 資料主題 | 問題類型 | 問題說明 | 處理狀態 | 相關筆記 |
|---|---|---|---|---|
| `[[data_topic]]` | missing_value / duplicate / outlier / format_error / encoding / unclear_definition / other |  | unresolved / workaround / solved / pending | `[[troubleshooting_note]]` |

### 常見資料品質問題

| 問題類型 | 說明 |
|---|---|
| missing_value | 缺失值、空值或資料不完整 |
| duplicate | 重複資料或重複紀錄 |
| outlier | 異常值或不合理資料 |
| format_error | 日期、數值、文字或欄位格式不一致 |
| encoding | 編碼、亂碼或語系問題 |
| unclear_definition | 欄位定義、資料來源或計算方式不清楚 |
| other | 其他資料品質問題 |

### 使用規則

- 若資料品質問題會反覆出現，應建立或更新 `wiki/troubleshooting/`。
- 若資料品質問題需要固定處理方式，應建立或更新 `wiki/workflows/`。
- 若資料品質問題涉及公式或計算方式，應更新 [[formula_index]]。

## 9. 資料處理與轉換追蹤

本區塊用於整理資料清理、轉換、格式整理或資料處理流程。

| 資料主題 | 處理方式 | 相關流程 | 使用工具 | 處理狀態 |
|---|---|---|---|---|
| `[[data_topic]]` | clean / transform / merge / filter / parse / validate / export / other | `[[workflow_name]]` | `[[tool_name]]` | not_started / partial / verified / needs_review |

### 使用規則

- 如果處理方式只是單次操作，可以寫在資料筆記中。
- 如果處理方式會重複使用，應建立或更新 `wiki/workflows/`。
- 如果處理方式依賴特定工具，應同步更新 [[tool_index]]。
- 如果處理方式涉及公式、模型或指標計算，應同步更新 [[formula_index]]。
- 如果處理方式可能影響任務或專案結果，應同步更新 [[task_index]] 或 [[project_index]]。

## 15. 維護規則

### 必須維護

- 新增 `wiki/data_notes/` 筆記後，必須更新 [[#3. 資料筆記總表]]。
- 新增資料筆記後，應更新 [[#5. 資料來源追蹤]]。
- 若資料可能敏感，必須更新 [[#6. 資料敏感性追蹤]]。
- 若資料涉及欄位或結構理解，應更新 [[#7. 資料結構與欄位追蹤]]。
- 若資料有品質問題，應更新 [[#8. 資料品質問題追蹤]]。
- 若資料涉及清理或轉換方式，應更新 [[#9. 資料處理與轉換追蹤]]。
- 若資料被任務、專案或流程使用，應更新 [[#10. 資料與任務 / 專案 / 流程對應]]。
- 若資料涉及公式、模型或指標計算，應更新 [[#11. 資料與公式 / 模型對應]] 與 [[formula_index]]。
- 若資料是近期重要學習內容，應更新 [[home]] 的近期學習重點。

### 不建議

- 不要把完整資料內容貼到本 index。
- 不要把本頁當成資料筆記本體。
- 不要在本頁保存客戶資料、保單資料、帳號、密碼、API Key、Token、完整內部系統網址或未遮蔽截圖。
- 不要保存完整內部資料表名稱、敏感欄位代碼或可反推內部系統架構的資訊。
- 不要在本頁建立完整 Knowledge Graph；完整關聯應放在正式 `wiki/data_notes/` 筆記中。
- 不要把尚未確認的欄位定義或資料邏輯寫成已確認事實。
