# Workflow Index

本頁用於整理 `wiki/workflows/` 中的流程筆記。

`workflow_index.md` 的重點不是保存完整流程內容，而是提供流程型筆記的導覽入口，幫助快速查詢：

- 目前整理過哪些流程
- 每個流程的輸入、處理步驟與輸出是什麼
- 流程使用哪些工具、資料、公式或概念
- 流程和哪些任務、專案或業務背景有關
- 流程是否已驗證、仍在草稿階段或需要確認
- 哪些流程可以被重複使用

---

## 1. Index 說明

本 index 對應以下資料夾：

| 類型 | 位置 | 說明 |
|---|---|---|
| 流程筆記 | `wiki/workflows/` | 實習期間整理出的操作流程、資料流程、報告流程、工具使用流程、開發流程與審查流程 |

適合放入 `wiki/workflows/` 的內容包括：

| 類型 | 範例 |
|---|---|
| 筆記整理流程 | 原始筆記匯入、daily log 產生、wiki 筆記整理、index 更新 |
| 工具操作流程 | Google ADK 報告產生流程、Codex 整理筆記流程、Obsidian 維護流程 |
| 資料處理流程 | 資料匯入、欄位清理、格式轉換、報表整理 |
| 開發流程 | 環境建立、測試、部署、版本更新 |
| 文件流程 | 報告產生、會議摘要整理、專案文件撰寫 |
| 審查流程 | PoC 審查、上線前確認、架構檢視、流程卡控的抽象化整理 |
| 錯誤排除流程 | 問題重現、原因排查、解法驗證、預防方式整理 |

## 5. 流程輸入與輸出

本區塊用於快速查看流程的 Input / Processing / Output。

| 流程 | Input | Processing | Output | 相關資料 |
|---|---|---|---|---|
| `[[workflow_name]]` |  |  |  | `[[data_note]]` |

### 使用規則

- Input 應描述流程需要什麼輸入，例如原始筆記、資料檔、API 回傳、使用者需求。
- Processing 應描述流程主要處理步驟，不需要寫完整細節。
- Output 應描述流程產出，例如 daily log、wiki note、report、code、data file。
- 若流程處理資料，應同步更新 [[data_index]]。
- 若流程涉及公式或計算方式，應同步更新 [[formula_index]]。

## 6. 流程與工具對應

本區塊用於查看流程使用哪些工具。

| 流程 | 工具 | 工具角色 | 使用狀態 |
|---|---|---|---|
| `[[workflow_name]]` | `[[tool_name]]` | input / processing / validation / output / documentation | planned / used / verified / replaced / pending |

### 工具角色說明

| 工具角色 | 說明 |
|---|---|
| input | 工具用於取得、匯入或整理輸入 |
| processing | 工具用於主要處理流程 |
| validation | 工具用於檢查、測試或驗證 |
| output | 工具用於產生結果、報告或檔案 |
| documentation | 工具用於記錄、整理或維護文件 |

### 使用規則

- 若流程使用新工具，應同步更新 [[tool_index]]。
- 若工具只是一次性使用，不一定需要建立流程。
- 若工具操作會重複執行，應建立或更新正式 workflow 筆記。

### 驗證狀態定義

| 狀態 | 說明 |
|---|---|
| not_tested | 尚未實際執行或測試 |
| partially_verified | 已部分執行，但仍有部分步驟待確認 |
| verified | 已完整執行，流程可重複使用 |
| needs_review | 流程可能涉及權限、敏感資訊、內部規則或業務邏輯，需要確認後才能定稿 |

### 優先度判斷

| 優先度 | 判斷標準 |

## 11. 流程到 Wiki 的延伸

有些流程筆記整理後，會需要延伸成其他正式 wiki 筆記。

| 流程 | 可延伸類型 | 建議位置 | 是否已建立 |
|---|---|---|---|
| `[[workflow_name]]` | tool / task / project / troubleshooting / data / formula / business_context / glossary | `wiki/...` | 是 / 否 / 待確認 |

### 延伸判斷

| 可延伸類型 | 判斷方式 |
|---|---|
| tool | 流程中使用到重要工具 |

| project | 流程屬於某個專案或 PoC |
| troubleshooting | 流程中出現可重複參考的錯誤或解法 |
| data | 流程處理特定資料來源、欄位或資料格式 |
| formula | 流程涉及公式、模型、指標或計算方式 |
| business_context | 流程涉及業務背景、審查、上線或組織流程 |
| glossary | 流程中包含縮寫、專有名詞或需要翻譯的術語 |

## 13. 維護規則

### 必須維護

- 新增 `wiki/workflows/` 筆記後，必須更新 [[#3. 流程總表]]。
- 新增流程筆記後，應更新 [[#5. 流程輸入與輸出]]。
- 若流程使用工具，應更新 [[#6. 流程與工具對應]] 與 [[tool_index]]。
- 若流程被任務或專案使用，應更新 [[#7. 流程與任務 / 專案對應]]。
- 若流程已驗證或待確認，應更新 [[#8. 流程驗證狀態]]。
- 若流程出現錯誤或限制，應更新 [[#9. 流程常見問題追蹤]] 與 [[troubleshooting_index]]。
- 若流程涉及資料、公式、業務背景或術語，應同步更新對應 index。
- 若流程是近期重要流程，應更新 [[home]] 的近期學習重點。

### 不建議

- 不要把完整流程步驟寫在本 index。
- 不要把本頁當成流程筆記本體。
- 不要在本頁保存客戶資料、帳號、密碼、API Key、Token、完整內部系統網址或未遮蔽截圖。
- 不要在本頁保存未抽象化的公司內部系統架構、審查細節、資料表名稱或敏感欄位。
- 不要在本頁建立完整 Knowledge Graph；完整關聯應放在正式 `wiki/workflows/` 筆記中。
- 不要把一次性、無後續價值的小操作都建立成高優先度流程。
