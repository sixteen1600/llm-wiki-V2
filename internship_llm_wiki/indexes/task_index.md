# Task Index

本頁用於整理 `wiki/tasks/` 中的任務筆記。

`task_index.md` 的重點不是保存完整任務內容，而是提供任務型筆記的導覽入口，幫助快速查詢：

- 目前有哪些任務
- 任務屬於哪個專案
- 任務目前狀態
- 任務使用哪些工具、流程或資料
- 任務是否已完成、卡住或需要確認
- 任務是否已產出正式 wiki 筆記或成果

---

## 1. Index 說明

本 index 對應以下資料夾：

| 類型 | 位置 | 說明 |
|---|---|---|
| 任務筆記 | `wiki/tasks/` | 實習期間的具體任務、待辦事項、交付內容、執行紀錄與成果追蹤 |

適合放入 `wiki/tasks/` 的內容包括：

| 類型 | 範例 |
|---|---|
| 實習任務 | 主管交辦的學習、整理、測試或開發任務 |
| 技術任務 | 建立環境、測試工具、撰寫程式、整理 API 使用方式 |
| 文件任務 | 撰寫報告、整理 wiki、製作流程文件 |
| 資料任務 | 觀察資料、整理欄位、測試資料處理流程 |
| 研究任務 | 了解工具、比較方案、整理概念與限制 |
| 專案子任務 | 某個 PoC、Demo、Prototype 或專案中的階段性工作 |

## 6. 卡住或待確認任務

本區塊整理目前無法繼續、需要確認、等待權限或等待外部資訊的任務。

| 任務 | 卡住原因 | 需要確認事項 | 確認對象 | 狀態 |
|---|---|---|---|---|
| `[[task_name]]` |  |  | 使用者 / 主管 / 同事 / 待確認 | blocked / pending / waiting_review |

### 常見卡住原因

| 類型 | 說明 |
|---|---|
| 權限問題 | 缺少系統、資料、工具或檔案存取權限 |
| 環境問題 | 本機、套件、版本、設定或執行環境不一致 |
| 需求不明 | 任務目標、輸入、輸出或驗收標準不清楚 |
| 資料問題 | 資料來源、欄位、格式或品質尚未確認 |
| 技術問題 | 工具、程式、API、流程或模型行為不如預期 |
| 審查問題 | 需要主管、同事或其他角色確認後才能繼續 |

## 13. 維護規則

### 必須維護

- 新增 `wiki/tasks/` 筆記後，必須更新 [[#3. 任務總表]]。
- 任務開始進行時，應更新 [[#4. 進行中任務]]。
- 任務完成後，應更新 [[#5. 已完成任務]] 與 [[#9. 任務成果追蹤]]。
- 任務卡住或需要確認時，應更新 [[#6. 卡住或待確認任務]]。
- 任務屬於某個專案時，應更新 [[#7. 任務與專案對應]] 與 [[project_index]]。
- 任務使用到工具、流程、資料或公式時，應同步更新對應 index。
- 任務產生錯誤排除經驗時，應同步更新 [[troubleshooting_index]]。

### 不建議

- 不要把完整任務內容寫在本 index。
- 不要把本頁當成任務筆記本體。
- 不要在本頁保存客戶資料、帳號、密碼、API Key、Token、完整內部系統網址或未遮蔽截圖。
- 不要在本頁建立完整 Knowledge Graph；完整關聯應放在正式 `wiki/tasks/` 筆記中。
- 不要長期保留已完成且無參考價值的任務在進行中區塊。
