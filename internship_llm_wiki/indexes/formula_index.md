# Formula Index

本頁用於整理 `wiki/formulas/` 中的公式、模型、指標與計算方式筆記。

`formula_index.md` 的重點不是保存完整公式推導內容，而是提供公式型筆記的導覽入口，幫助快速查詢：

- 目前整理過哪些公式、模型或計算方式
- 每個公式的用途、輸入、輸出與使用情境
- 公式和哪些資料、任務、專案、流程或工具有關
- 公式是否涉及敏感計算邏輯，需要抽象化或確認
- 哪些公式仍需要補充符號說明、驗證方式或範例計算

## 1. Index 說明

本 index 對應以下資料夾：

| 類型 | 位置 | 說明 |
|---|---|---|
| 公式筆記 | `wiki/formulas/` | 實習期間接觸到的數學公式、統計公式、模型定義、損失函數、評分公式、權重計算與指標計算方式 |

適合放入 `wiki/formulas/` 的內容包括：

| 類型 | 範例 |
|---|---|
| 數學公式 | 加權平均、比例計算、成長率、標準化公式 |
| 統計公式 | 平均數、變異數、標準差、相關係數、檢定統計量 |
| 模型定義 | 評分模型、分類模型、預測模型、決策模型的抽象化定義 |
| 損失函數 | MSE、Cross Entropy、其他模型訓練目標 |
| 指標計算 | Accuracy、Precision、Recall、F1、命中率、覆蓋率 |
| 權重計算 | 權重分配、加權分數、時間衰減、排序分數 |
| 業務計算 | 報表指標、評估分數、流程衡量指標的抽象化版本 |

## 5. 公式來源追蹤

本區塊用於追蹤公式、模型或計算方式最早出現在哪些原始筆記或學習紀錄中。

| 公式 | 首次出現日期 | 原始來源 | 對應 Log | 整理狀態 |
|---|---|---|---|---|
| `[[formula_name]]` | YYYY-MM-DD | `raw/imported_notes/YYYY/YYYY_MM_DD_Ddd.md` | `logs/daily/YYYY-MM-DD.md` | not_started / partial / completed |

### 使用規則

- 若公式來自每日原始筆記，應填入 `raw/imported_notes/` 來源。
- 若公式已整理成 daily log，應填入對應 `logs/daily/`。
- 若公式來自會議整理，應填入 `logs/meetings/`。
- 不要在本 index 中保存完整敏感公式內容。
- 若公式涉及公司內部計算規則，只保留抽象化描述與來源位置。

## 6. 公式敏感性追蹤

本區塊用於檢查公式、模型或計算方式是否涉及敏感資訊。

| 公式 | 敏感性 | 風險類型 | 處理方式 | 確認狀態 |
|---|---|---|---|---|
| `[[formula_name]]` | none / abstracted / needs_review / sensitive | business_rule / scoring_logic / model_rule / internal_metric / data_dependency / other | remove / abstract / simplify / keep / pending | unchecked / reviewed / confirmed |

### 敏感性定義

| 敏感性 | 說明 |
|---|---|
| none | 不含敏感資訊，可直接整理 |
| abstracted | 原始公式可能敏感，但已抽象化處理 |
| needs_review | 是否可保存仍需確認 |
| sensitive | 明確涉及敏感計算邏輯，不應完整保存在 wiki 中 |

### 常見風險類型

| 風險類型 | 說明 |
|---|---|
| business_rule | 未公開的業務規則或決策邏輯 |
| scoring_logic | 評分、排序、分級或篩選邏輯 |
| model_rule | 模型判斷規則、模型參數或內部策略 |
| internal_metric | 公司內部指標或報表計算方式 |
| data_dependency | 公式依賴敏感資料欄位或內部資料來源 |
| other | 其他需確認或抽象化的資訊 |

### 使用規則

- Input 應描述公式需要哪些輸入，例如數值欄位、參數、資料表、模型輸入或人工設定值。

## 11. 公式驗證狀態

本區塊用於追蹤公式、模型或計算方式是否已確認與驗證。

| 公式 | 驗證狀態 | 驗證方式 | 驗證日期 | 待確認事項 |
|---|---|---|---|---|
| `[[formula_name]]` | not_verified / partially_verified / verified / needs_review |  | YYYY-MM-DD |  |

### 驗證狀態定義

| 狀態 | 說明 |
|---|---|
| not_verified | 尚未確認公式是否正確 |
| partially_verified | 已確認部分定義或部分計算流程，但仍有待確認項目 |
| verified | 已確認公式定義、符號、輸入、輸出與使用情境 |
| needs_review | 涉及敏感資訊、內部規則、業務邏輯或模型設定，需要確認後才能定稿 |

### 常見驗證方式

| 驗證方式 | 說明 |
|---|---|
| manual_check | 人工檢查公式、符號與說明是否一致 |
| sample_calculation | 使用假資料或抽象資料進行範例計算 |
| source_check | 回到原始筆記、文件或會議紀錄確認來源 |
| code_check | 與程式碼或腳本中的計算方式比對 |
| peer_review | 由主管、同事或相關角色確認 |
| needs_review | 目前仍待確認，不應定稿 |

## 15. 維護規則

### 必須維護

- 新增 `wiki/formulas/` 筆記後，必須更新 [[#3. 公式筆記總表]]。
- 新增公式筆記後，應更新 [[#5. 公式來源追蹤]]。
- 若公式可能敏感，必須更新 [[#6. 公式敏感性追蹤]]。
- 若公式涉及重要符號或變數，應更新 [[#7. 符號與變數追蹤]]。
- 若公式有明確輸入與輸出，應更新 [[#8. 公式輸入與輸出]]。
- 若公式使用資料，應更新 [[#9. 公式與資料對應]] 與 [[data_index]]。
- 若公式被任務、專案或流程使用，應更新 [[#10. 公式與任務 / 專案 / 流程對應]]。
- 若公式完成確認或仍需檢查，應更新 [[#11. 公式驗證狀態]]。
- 若公式是近期重要學習內容，應更新 [[home]] 的近期學習重點。

### 不建議

- 不要把完整敏感公式或內部計算邏輯貼到本 index。
- 不要把本頁當成公式筆記本體。
- 不要在本頁保存客戶資料、保單資料、帳號、密碼、API Key、Token、完整內部系統網址或未遮蔽截圖。
- 不要保存完整內部資料表名稱、敏感欄位代碼或可反推內部決策規則的資訊。
- 不要在本頁建立完整 Knowledge Graph；完整關聯應放在正式 `wiki/formulas/` 筆記中。
- 不要把尚未確認的公式、符號定義或計算邏輯寫成已確認事實。
