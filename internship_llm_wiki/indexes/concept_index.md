# Concept Index

本頁用於整理 `wiki/concepts/` 中的概念筆記。

`concept_index.md` 的重點不是保存完整概念內容，而是提供概念型知識的導覽入口，幫助快速查詢：

- 已整理過哪些概念
- 每個概念的中文名稱、英文名稱與縮寫
- 概念第一次出現在哪些學習紀錄中
- 概念和哪些工具、流程、任務、專案、資料、公式或業務背景有關
- 哪些概念仍需要補充、確認或延伸成其他 wiki 筆記

## 1. Index 說明

本 index 對應以下資料夾：

| 業務相關概念 | 審查、上線、卡控、流程治理 |

## 5. 概念來源追蹤

本區塊用於追蹤概念最早出現在哪些原始筆記或學習紀錄中。

| 概念 | 首次出現日期 | 原始來源 | 對應 Log | 整理狀態 |
|---|---|---|---|---|
| `[[concept_name]]` | YYYY-MM-DD | `raw/imported_notes/YYYY/YYYY_MM_DD_Ddd.md` | `logs/daily/YYYY-MM-DD.md` | not_started / partial / completed |

### 使用規則

- 若概念來自每日原始筆記，應填入 `raw/imported_notes/` 來源。
- 若概念已整理成 daily log，應填入對應 `logs/daily/`。
- 若概念是從會議整理中萃取，應填入 `logs/meetings/`。
- 不要在本 index 中複製完整原始筆記內容。
- 若來源內容涉及敏感資訊，僅保留抽象化描述與檔案位置。

## 6. 概念關聯追蹤

本區塊用於概略追蹤概念與其他 wiki 類型的關係。

完整的 Knowledge Graph 關係應維護在各個正式 wiki 筆記的 `Knowledge Graph 關聯` 區塊中。  
本區塊只做索引用的摘要，不取代正式 wiki 筆記內的關聯維護。

| 概念 | 相關工具 | 相關流程 | 相關任務 | 相關專案 | 相關資料 | 相關公式 | 業務背景 |
|---|---|---|---|---|---|---|---|
| `[[concept_name]]` |  |  |  |  |  |  |  |

### 關聯填寫原則

- 優先使用 Obsidian 雙向連結，例如 `[[tool_name]]`。
- 只放最重要的關聯，不要把所有弱相關頁面都列入。
- 若有明確上下游關係，應回到該概念筆記的 `Knowledge Graph 關聯` 區塊中維護。
- 本區塊只作快速導覽，不作完整 KG 資料庫。

## 7. 待補概念

| 概念 | 來源 | 建議檔案 | 優先度 | 需要補充的內容 |
|---|---|---|---|---|
|  | `logs/daily/YYYY-MM-DD.md` | `wiki/concepts/concept_name.md` | 高 / 中 / 低 |  |

### 優先度判斷

| 優先度 | 判斷標準 |
|---|---|
| 高 | 任務核心、專案核心、反覆出現、主管要求、容易混淆或容易影響後續工作 |
| 中 | 有學習價值，可能之後會用到，但目前不是主要任務核心 |
| 低 | 輔助背景知識，短期內不影響任務執行 |

## 9. 概念到 Wiki 的延伸

有些概念整理後，會需要延伸成其他正式 wiki 筆記。

| 概念 | 可延伸類型 | 建議位置 | 是否已建立 |
|---|---|---|---|
| `[[concept_name]]` | tool / workflow / task / project / data / formula / business_context / glossary | `wiki/...` | 是 / 否 / 待確認 |

### 延伸判斷

| 可延伸類型 | 判斷方式 |
|---|---|
| tool | 概念牽涉具體工具、平台、套件或系統 |
| workflow | 概念已形成可重複執行的流程 |
| task | 概念對應到具體任務或待辦 |
| project | 概念擴展成較大型專案、PoC 或原型 |
| data | 概念涉及資料來源、欄位、格式或資料品質 |
| formula | 概念涉及公式、模型、指標或計算方式 |
| business_context | 概念與業務流程、組織流程或審查流程有關 |
| glossary | 概念包含縮寫、英文術語或中文翻譯 |

## 11. 維護規則

### 必須維護

- 新增 `wiki/concepts/` 筆記後，必須更新 [[#3. 概念總表]]。
- 新增概念筆記後，應更新 [[#5. 概念來源追蹤]]。
- 若概念與工具、流程、任務、專案、資料、公式或業務背景有關，應更新 [[#6. 概念關聯追蹤]]。
- 若某概念尚未整理完成，應更新 [[#7. 待補概念]]。
- 若某概念容易與其他概念混淆，應更新 [[#8. 容易混淆的概念]]。
- 若概念需要延伸成其他 wiki 筆記，應更新 [[#9. 概念到 Wiki 的延伸]]。
- 若新增縮寫或英文術語，應同步更新 [[glossary_index]]。

### 不建議

- 不要把完整概念說明寫在本 index。
- 不要把本頁當作正式概念筆記。
- 不要在本頁保存客戶資料、帳號、密碼、API Key、Token、完整內部系統網址或未遮蔽截圖。
- 不要在本頁建立完整 Knowledge Graph；完整關聯應放在正式 `wiki/concepts/` 筆記中。
- 不要把所有弱相關概念都列入關聯表，避免 index 過度膨脹。
