# Business Context Index

本頁用於整理 `wiki/business_context/` 中的業務背景、組織流程、審查流程與實習情境脈絡筆記。

`business_context_index.md` 的重點不是保存完整內部流程細節，而是提供業務脈絡型筆記的導覽入口，幫助快速查詢：

- 目前整理過哪些業務背景或組織流程
- 這些背景和哪些任務、專案、工具或流程有關
- 哪些內容屬於業務流程、審查流程、上線流程或組織角色
- 哪些內容需要抽象化，避免保存公司內部敏感資訊
- 哪些業務背景應延伸成概念、流程、任務、專案或術語筆記

---

## 1. Index 說明

本 index 對應以下資料夾：

| 類型 | 位置 | 說明 |
|---|---|---|
| 業務背景筆記 | `wiki/business_context/` | 實習期間接觸到的業務流程、組織流程、審查流程、上線流程、部門脈絡與工作情境背景 |

適合放入 `wiki/business_context/` 的內容包括：

| 類型 | 範例 |
|---|---|
| 實習情境背景 | 任務來源、主管期待、部門工作方式、文件產出情境 |
| 業務流程 | 報告流程、資料流轉流程、需求確認流程、文件審查流程 |
| 組織流程 | 專案啟動前檢視、架構檢視、跨角色協作、上線前確認 |
| 審查流程 | PoC 審查、架構審查、文件審查、上線前檢核 |
| 上線流程 | 系統建置後檢核、上線前卡控、變更確認、風險確認 |
| 角色脈絡 | 主管、同事、架構師、審查角色、業務窗口的抽象化說明 |
| 公司情境抽象化 | 公司內部流程的非敏感、可學習版本 |

不適合直接放入本 index 的內容：

| 不適合內容 | 應放位置 |
|---|---|
| 完整技術概念 | `wiki/concepts/` |
| 工具使用方式 | `wiki/tools/` |
| 可重複執行的操作步驟 | `wiki/workflows/` |
| 具體任務執行紀錄 | `wiki/tasks/` |
| 大型 PoC 或 Demo 專案 | `wiki/projects/` |
| 資料欄位或資料結構 | `wiki/data_notes/` |
| 公式或計算方式 | `wiki/formulas/` |
| 縮寫與名詞定義 | `wiki/glossary/` |

## 5. 來源追蹤

本區塊用於追蹤業務背景最早出現在哪些原始筆記或學習紀錄中。

| 業務背景 | 首次出現日期 | 原始來源 | 對應 Log | 整理狀態 |
|---|---|---|---|---|
| `[[business_context_name]]` | YYYY-MM-DD | `raw/imported_notes/YYYY/YYYY_MM_DD_Ddd.md` | `logs/daily/YYYY-MM-DD.md` | not_started / partial / completed |

### 使用規則

- 若業務背景來自每日原始筆記，應填入 `raw/imported_notes/` 來源。
- 若業務背景已整理成 daily log，應填入對應 `logs/daily/`。
- 若業務背景來自會議整理，應填入 `logs/meetings/`。
- 不要在本 index 中複製完整會議內容。
- 若來源包含公司內部流程、組織細節或系統資訊，應使用抽象化描述。

## 6. 敏感性與抽象化追蹤

本區塊用於檢查業務背景是否涉及公司內部敏感資訊。

| 業務背景 | 敏感性 | 風險類型 | 處理方式 | 確認狀態 |
|---|---|---|---|---|
| `[[business_context_name]]` | none / abstracted / needs_review / sensitive | internal_process / organization_detail / system_detail / approval_rule / business_rule / role_detail / other | remove / abstract / generalize / keep / pending | unchecked / reviewed / confirmed |

### 敏感性定義

| 敏感性 | 說明 |
|---|---|
| none | 不含敏感資訊，可直接整理成抽象業務背景 |
| abstracted | 原始內容可能敏感，但已抽象化處理 |
| needs_review | 是否可保存仍需確認 |
| sensitive | 明確涉及敏感資訊，不應完整保存在 wiki 中 |

### 常見風險類型

| 風險類型 | 說明 |
|---|---|
| internal_process | 公司內部流程細節、審查細節、上線細節 |
| organization_detail | 具體組織、人員、權責或內部協作細節 |
| system_detail | 內部系統名稱、架構、網址、權限或串接資訊 |
| approval_rule | 未公開審查標準、核准條件或卡控邏輯 |
| business_rule | 未公開業務規則、決策邏輯或計算方式 |
| role_detail | 可識別個人或具體職務分工的細節 |
| other | 其他需確認或抽象化的資訊 |

### 抽象化原則

| 原始內容類型 | 建議抽象化方式 |
|---|---|
| 具體組織名稱 | 改成「審查單位」、「架構檢視角色」、「流程確認角色」 |
| 具體人名 | 改成「主管」、「同事」、「架構師」、「業務窗口」 |
| 內部系統名稱 | 改成「內部系統」、「目標系統」、「來源系統」 |
| 內部審查規則 | 改成「審查標準」、「確認項目」、「上線前檢核」 |
| 內部流程細節 | 保留概念，不保留可執行的完整內部步驟 |

## 7. 業務背景與任務 / 專案對應

本區塊用於查看業務背景支援或影響哪些任務與專案。

| 業務背景 | 相關任務 | 相關專案 | 關聯角色 | 影響程度 |
|---|---|---|---|---|
| `[[business_context_name]]` | `[[task_name]]` | `[[project_name]]` | background / requirement / review / approval / constraint / output_context | low / medium / high |

### 關聯角色說明

| 關聯角色 | 說明 |
|---|---|
| background | 提供理解任務或專案的背景 |
| requirement | 形成任務或專案需求來源 |
| review | 與審查、檢視或確認流程有關 |
| approval | 與核准、上線或卡控有關 |
| constraint | 對任務或專案形成限制條件 |
| output_context | 說明報告、文件或系統輸出的使用情境 |

### 使用規則

- 若背景支援某個任務，應同步更新 [[task_index]]。
- 若背景屬於某個專案脈絡，應同步更新 [[project_index]]。
- 若背景影響任務或專案方向，應在正式筆記中補充 Knowledge Graph 關聯。
- 不要在此保存完整內部審查規則。

## 10. 審查與上線流程追蹤

本區塊用於整理與審查、檢核、卡控、上線前確認相關的背景。

| 背景 | 流程類型 | 目的 | 相關專案 / 流程 | 狀態 |
|---|---|---|---|---|
| `[[business_context_name]]` | review / architecture_review / approval / deployment_check / change_control / other |  |  | draft / active / reviewed / needs_review |

### 常見流程類型

| 流程類型 | 說明 |
|---|---|
| review | 一般審查、檢視或確認流程 |
| architecture_review | 架構、系統串接或技術方案檢視 |
| approval | 核准、同意或決策確認 |
| deployment_check | 上線前檢核、部署確認、風險確認 |
| change_control | 變更管理、變更審查、上線卡控 |
| other | 其他需抽象化整理的流程 |

### 使用規則

- 只記錄抽象化後的審查目的與流程脈絡。
- 不保存完整內部審查細節、核准條件或上線規則。
- 若流程可重複執行，應建立或更新 `wiki/workflows/`。
- 若背景與某個專案高度相關，應同步更新 [[project_index]]。
- 若涉及專有縮寫，應同步更新 [[glossary_index]]。

| data | 背景涉及資料來源、資料流或資料欄位 |
| formula | 背景涉及公式、指標、模型或計算方式 |

## 14. 維護規則

### 必須維護

- 新增 `wiki/business_context/` 筆記後，必須更新 [[#3. 業務背景總表]]。
- 新增業務背景筆記後，應更新 [[#5. 來源追蹤]]。
- 若業務背景可能敏感，必須更新 [[#6. 敏感性與抽象化追蹤]]。
- 若業務背景與任務或專案有關，應更新 [[#7. 業務背景與任務 / 專案對應]]。
- 若業務背景涉及工具或流程，應更新 [[#8. 業務背景與工具 / 流程對應]]。
- 若業務背景涉及角色、責任或協作方式，應更新 [[#9. 角色與責任追蹤]]。
- 若業務背景涉及審查、檢核、卡控或上線流程，應更新 [[#10. 審查與上線流程追蹤]]。
- 若業務背景包含縮寫或專有名詞，應同步更新 [[glossary_index]]。
- 若業務背景是近期重要學習內容，應更新 [[home]] 的近期學習重點。

### 不建議

- 不要把完整內部流程、完整會議紀錄或完整審查規則寫在本 index。
- 不要把本頁當成業務背景筆記本體。
- 不要在本頁保存客戶資料、保單資料、帳號、密碼、API Key、Token、完整內部系統網址或未遮蔽截圖。
- 不要保存未抽象化的公司內部組織架構、審查細節、系統串接細節、資料表名稱或敏感欄位。
- 不要在本頁建立完整 Knowledge Graph；完整關聯應放在正式 `wiki/business_context/` 筆記中。
- 不要把尚未確認的流程、角色或審查邏輯寫成已確認事實。
