# Tool Index

本頁用於整理 `wiki/tools/` 中的工具筆記。

`tool_index.md` 的重點不是保存完整工具使用教學，而是提供工具型筆記的導覽入口，幫助快速查詢：

- 目前接觸過哪些工具、平台、套件或系統
- 每個工具的用途是什麼
- 工具與哪些任務、專案、流程、資料或問題有關
- 工具目前是否已會使用、仍在學習或需要補充
- 工具是否產生可複用的操作流程、錯誤排除或範例

## 6. 工具與任務 / 專案對應

| 工具 | 相關任務 | 相關專案 | 工具角色 | 使用狀態 |
|---|---|---|---|---|
| `[[tool_name]]` | `[[task_name]]` | `[[project_name]]` | main_tool / supporting_tool / testing_tool / documentation_tool | planned / used / replaced / pending |

### 工具角色說明

| 工具角色 | 說明 |
|---|---|
| main_tool | 任務或專案的主要工具 |
| supporting_tool | 輔助處理、查詢、整理或測試用工具 |
| testing_tool | 用於測試、驗證、PoC 或實驗 |
| documentation_tool | 用於筆記、報告、文件或 wiki 整理 |

### 使用規則

- 若工具用於某個任務，應同步更新 [[task_index]]。
- 若工具用於某個專案，應同步更新 [[project_index]]。
- 若工具形成固定操作流程，應同步更新 [[workflow_index]]。
- 若工具產生錯誤排除經驗，應同步更新 [[troubleshooting_index]]。

## 9. 待建立工具筆記

本區塊整理已經出現在 log 中，但尚未建立正式 `wiki/tools/` 筆記的工具。

| 工具 | 來源 | 建議檔案 | 優先度 | 需要補充的內容 |
|---|---|---|---|---|

### 優先度判斷

| 優先度 | 判斷標準 |
|---|---|
| 高 | 任務核心、專案核心、主管要求、反覆使用、容易出錯、需要形成操作流程 |
| 中 | 有學習價值，可能之後會使用，但目前不是主要任務核心 |
| 低 | 只是輔助工具，短期內不影響主要任務執行 |

## 12. 維護規則

### 必須維護

- 新增 `wiki/tools/` 筆記後，必須更新 [[#3. 工具總表]]。
- 新增工具筆記後，應更新 [[#5. 工具學習狀態]]。
- 若工具被任務或專案使用，應更新 [[#6. 工具與任務 / 專案對應]]。
- 若工具參與固定流程，應更新 [[#7. 工具與流程對應]] 與 [[workflow_index]]。
- 若工具出現錯誤或限制，應更新 [[#8. 工具常見問題追蹤]] 與 [[troubleshooting_index]]。
- 若工具涉及資料、公式或術語，應同步更新對應 index。
- 若工具是近期重要工具，應更新 [[home]] 的近期學習重點。

### 不建議

- 不要把完整工具教學寫在本 index。
- 不要把本頁當成工具筆記本體。
- 不要在本頁保存帳號、密碼、API Key、Token、完整內部系統網址、未遮蔽截圖或客戶資料。
- 不要在本頁保存未抽象化的公司內部系統細節。
- 不要在本頁建立完整 Knowledge Graph；完整關聯應放在正式 `wiki/tools/` 筆記中。
- 不要把一次性、無後續價值的小工具都列成高優先度。
