# examples/

This folder contains reference examples for the internship LLM Wiki workflow.

These files are not production notes. They are examples for Codex to learn the expected structure, tone, level of abstraction, source tracing, and sensitive information handling.

## Folder Structure

| Folder | Purpose |
|---|---|
| `raw_notes/` | Example raw imported notes |
| `daily_logs/` | Example daily logs generated from raw notes |
| `meeting_notes/` | Example meeting notes |
| `weekly_summaries/` | Example weekly summaries |
| `monthly_reviews/` | Example monthly reviews |
| `wiki_notes/` | Example wiki notes by category |
| `codex_requests/` | Example Codex request files generated by scripts |

## Rules

- Do not treat examples as real work records.
- Do not store client data, policy data, passwords, API keys, tokens, internal URLs, or unmasked screenshots.
- Keep examples abstract.
- Use examples only as few-shot references for Codex.
- Real work should go to `raw/`, `logs/`, `wiki/`, and `indexes/`.

## Recommended Use

When Codex is unsure about format, provide one example from this folder together with the relevant prompt.

For example:

```text
Use prompts/02_generate_daily_log.md.
Refer to examples/daily_logs/2026-07-03_example.md for formatting style.
Process raw/imported_notes/YYYY/YYYY_MM_DD_Ddd.md.
```
